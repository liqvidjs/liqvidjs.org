# FAQ {#faq}

<dl class="qa">
  <dt>Do liquid videos work on mobile?</dt>
  <dd>There are some UX improvements to be made, but yes, liquid videos should be fully functional on mobile. Check the <a href="/docs/guide/mobile">Mobile section of the Guide</a> for tips on making sure your videos are mobile-friendly.</dd>

  <dt>Can liquid videos be uploaded to YouTube/Twitter/etc.?</dt>
  <dd>No, you will have to host them yourself. But you can easily <a href="/docs/cli/render">make an MP4 recording of a Liqvid project</a> and upload that to social media for discoverability, and then provide a link to the interactive version.</dd>

  <dt>How does Liqvid compare to <a href="https://www.remotion.dev">Remotion</a>?</dt>
  <dd>
    <p>Overall, Liqvid is mainly focused on in-browser interactive videos, especially educational content/tutorials, while Remotion is more focused on generating static mp4's programmatically. That said, Liqvid has a <a href="/docs/cli/render"><code>render</code></a> command for generating static videos, and Remotion has the <a href="https://www.remotion.dev/docs/player"><code>@remotion/player</code></a> package for in-browser videos.</p>
    {/* .. */}
    <p>Things Remotion has that Liqvid doesn't:</p>
    <ul>
      <li>pretty timeline GUI</li>
      <li>utilities for <a href="https://www.remotion.dev/docs/audio-visualization">visualizing audio</a></li>
      <li>more animation helpers</li>
      <li>AWS Lambda integration</li>
      <li><a href="https://www.remotion.dev/docs/gif">synchronize with GIFs</a></li>
      <li>interactive player directly in page <sup class="footnote"><a href="#note1">[1]</a></sup></li>
      <li>paid support</li>
    </ul>
    {/* */}
    <p>Things Liqvid has that Remotion doesn't:</p>
    <ul>
      <li><a href="/docs/guide/packages">plugin suite</a> for recording audio voiceover, cursor motion, coding, and handwriting <sup class="footnote"><a href="#note2">[2]</a></sup></li>
      <li>customizable keyboard shortcuts</li>
      <li>thumbnail previews on the scrubber bar</li>
      <li>closed captions</li>
      <li>use the <a href="/docs/guide/animation#web-animations-api">Web Animations API</a> in videos</li>
      <li>MIT license</li>
    </ul>
  </dd>
</dl>

#### Notes

<ol class="footnotes">
  <li id="note1">Liqvid videos are embedded via <code>&lt;iframe&gt;</code>s, like YouTube videos. Putting a player directly in a page will be possible in v2.2.</li>
  <li id="note2">The next iteration of Liqvid recording plugins will be compatible with Remotion, as well as <a href="https://greensock.com/gsap/">GSAP</a> and plain HTML. (I really <em>really</em> <strong>really</strong> <strong><em>really</em></strong> want to avoid a standards war in the interactive video space.)</li>
</ol>
